The "Content" and "Sidebar" sections are inside a container div. This container changes to "display: flex" when the viewport hits the first break point (min-width: 480px).

Break Points

Mobile Devices

My default CSS rules are applied from 0 pixels and up, without Media Queries or Break Points, so the layout is Mobile First.

Content and Sidebar will be stacked.

Tablets

My first break point is set to min-width: 480px.

It applies some CSS rules only from 480 pixels width and up, to accomodate tablet screens.

The container div for "Content" and "Sidebar" sets it's display to flex, so it's children divs are arranged in a row (using Flexbox).

Computers

My second break point is set to min-width: 900px to accomodate computer screens.

The containr will get a fixed width and will be centered on the screen. There will be margins to the left and right of the container.

Hidden Alert

The rules form the break points that appear later in the document can override the ones that appear earlier (higher) in the document (because of the CSS cascade). Try to shrink the screen to less than 250 pixels wide.
